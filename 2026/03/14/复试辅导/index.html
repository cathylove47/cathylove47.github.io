🎯 递推策略（教学重点）

错排问题是组合数学中的经典模型，具有非常固定的递推公式。假设 n 个元素的错排数为 D(n)：

  1. 基本边界： 当 n=1 时，无法错排，即 D(1)=0；当 n=2 时，只有一种交换方式，即 D(2)=1。

  2. 状态转移方程： 对于第 n 个元素，它可以放在前 n−1 个位置的任意一个（共有 n−1 种选择）。假设它放在了第 k 个位置，那么原本在第 k 个位置的元素有两种情况：

    • 放到第 n 个位置：剩下的 n−2 个元素进行错排，方案数为 D(n−2)。
    • 不放到第 n 个位置：这相当于剩下的 n−1 个元素进行错排，方案数为 D(n−1)。

因此，核心递推公式为：D(n)=(n−1)×(D(n−1)+D(n−2))。

🎯 贪心与数学逻辑（教学重点）

这道题的核心在于数学分析和贪心思想。要在数组中选出三个数使其乘积最小，实际上只有两种组合可能：

  1. 三个最小的数相乘（例如数组全为正数，或者全为负数时）。
  2. 一个最小的数（即绝对值最大的负数）与两个最大的数相乘（例如数组中有正有负，通过“负 × 正 × 正”得负来获取极小值）。

解题思路： 假设将数组按从小到大排序为 A1​,A2​,…,An​：

  • 组合一的乘积为：A1​×A2​×A3​
  • 组合二的乘积为：A1​×An−1​×An​ 最终的最小乘积即为这二者中的最小值：min(A1​×A2​×A3​,A1​×An−1​×An​)。

进阶优化： 并不需要对整个数组进行 O(NlogN) 的完整排序，只需在单次 O(N) 的遍历中，利用几个变量分别维护并找出数组的最小的三个数和最大的两个数即可。

🎯 栈的典型应用（教学重点）

这道题是数据结构中**栈（Stack）的绝对经典应用，也是考研 408 数据结构大题的常客。核心思想是利用栈“后进先出”（LIFO）**的特性来处理配对问题。

解题思路：

  1. 遍历字符串： 从左到右依次读取每个字符。

  2. 遇到左括号： (、[、{ 时，将其压入栈中。

  3. 遇到右括号： )、]、} 时，需要与栈顶元素进行匹配：

    • 如果栈为空，说明没有对应的左括号，直接返回无效。
    • 弹出栈顶元素，检查是否与当前右括号类型匹配（例如 ( 匹配 )）。如果不匹配，返回无效。

  4. 遍历结束： 如果栈为空，说明所有左括号都成功闭合，返回有效；如果栈不为空（例如输入 ((），说明有未闭合的左括号，返回无效。

进阶优化： 如果字符串的长度为奇数，它肯定无法完全配对，可以在一开始就直接判断并返回无效（剪枝操作）。

🎯 高精度加法（教学重点）

这道题是 408 机试和秋招面试中极其经典的**“高精度计算（大数运算）”**入门题。当数字大小超过 long long 的极限（大约 19 位数字）时，我们必须依靠字符串或数组来模拟人类手工进行竖式加法的过程。

解题思路：

  1. 双指针从后往前： 设置两个指针分别指向 num1 和 num2 的末尾（即个位），从右向左按位相加。
  2. 处理进位 (Carry)： 每次相加时，除了两个字符对应的数字外，还要加上上一位的进位 carry（初始为 0）。当前位的实际结果是 (n1 + n2 + carry) % 10，新的进位是 (n1 + n2 + carry) / 10。
  3. 补齐短字符串： 如果其中一个字符串已经遍历完，可以将其对应的数字当作 0 继续参与运算，直到另一个字符串也遍历完。
  4. 处理最高位进位： 两个字符串都遍历完后，如果 carry 还不为 0（比如 9+1=10），必须在结果的最前面再追加一个 1。
  5. 反转结果： 因为我们是从个位开始计算并依次保存的，所以最后得到的结果是反的，输出前需要将生成的字符串反转。

中心扩展法解题思路：

🎯 贪心算法（教学重点）

这道题是考察贪心思想的绝佳范例。不要被“跳跃”的各种组合情况吓到，也不需要用复杂的深度优先搜索（DFS）去枚举每一种跳法，那样必定会超时。

解题思路（维护最远可达位置）：

  1. 核心逻辑： 我们只需要关心**“我当前能到达的最远位置是多少”**。如果这个最远位置已经覆盖（大于等于）了数组的最后一个下标，说明一定有办法跳过去。

  2. 一次遍历： 维护一个变量 max_reach（初始为 0）。遍历整个数组：

    • 前提条件检查： 如果当前遍历到的下标 i 已经超过了 max_reach，说明连当前位置都无法到达，直接返回 false（死胡同，比如遇到了跨不过去的 0）。
    • 更新最远距离： 如果当前位置可达，那么从当前位置出发，能到达的最远位置就是 i + nums[i]。我们尝试用它来更新 max_reach = max(max_reach, i + nums[i])。
    • 提前返回： 在遍历过程中，一旦发现 max_reach >= n - 1，就可以提前宣告胜利，返回 true。

🎯 贪心算法与双指针（教学重点）

这是贪心算法中最经典、最基础的“分配问题”。核心思想是好钢用在刀刃上：为了尽可能多地满足孩子，我们应该优先满足胃口最小的孩子，并且给他刚好能满足他的最小饼干。

解题思路：

  1. 排序： 首先将孩子的胃口数组 g 和饼干的尺寸数组 s 都按照从小到大的顺序进行排序。这是贪心策略能生效的前提。

  2. 双指针遍历： - 设置一个指针 child 指向 g 的开头，另一个指针 cookie 指向 s 的开头。

    • 比较当前饼干 s[cookie] 和当前孩子 g[child]：

      • 如果 s[cookie] >= g[child]，说明这块饼干能满足这个孩子，两个指针同时后移（child++, cookie++）。
      • 如果 s[cookie] < g[child]，说明这块饼干太小了，连当前胃口最小的孩子都满足不了，那它肯定也满足不了后面的孩子。直接废弃这块饼干，尝试下一块（仅 cookie++）。

  3. 结束条件： 当任意一个数组被遍历完时，循环结束。此时 child 指针的值恰好就是被满足的孩子数量。

这道题是字符串处理和贪心算法结合的完美案例。要保证“同一字母最多出现在一个片段中”，核心在于找到当前片段中所有字母的最远出现位置。

解题思路：

  1. 第一次遍历（记录最后出现位置）： - 既然一个字母必须完整地包含在一个片段里，我们就需要知道它最后一次出现在哪里。

    • 从头遍历字符串。对于遇到的每一个字符 s[i]，查表得到它的最后出现位置，并用它来刷新当前片段的边界：end = max(end, last_pos[s[i] - 'a'])。

    • 记录当前片段长度 end - start + 1，并将 start 更新为 end + 1，继续寻找下一个片段。

🎯 动态规划基础（教学重点）

这道题是学习**动态规划（Dynamic Programming）**的完美入门题。它的本质是一个斐波那契数列（Fibonacci sequence）。

解题思路：

  1. 定义状态： 假设 dp[i] 表示爬到第 i 阶楼梯的方法数。

  2. 寻找状态转移方程： 思考一下，你怎么才能到达第 i 阶？

    • 你只能从第 i-1 阶跨 1 步上来。
    • 或者从第 i-2 阶跨 2 步上来。 所以，到达第 i 阶的方法数，就等于到达第 i-1 阶的方法数加上到达第 i-2 阶的方法数。即：dp[i] = dp[i-1] + dp[i-2]。

  3. 初始化边界条件：

    • 爬 1 阶只有 1 种方法：dp[1] = 1
    • 爬 2 阶有 2 种方法：dp[2] = 2

  4. 空间复杂度优化（滚动数组思想）： 在计算 dp[i] 时，我们只需要用到 dp[i-1] 和 dp[i-2] 这两个数字，不需要把之前所有的结果都存下来。因此，可以用两个变量 prev1 和 prev2 来交替滚动更新，把空间复杂度从 O(N) 降维到 O(1)。

🎯 动态规划与组合数学（教学重点）

这道题是爬楼梯（Climbing Stairs）的二维升级版。它既可以用动态规划求解，也可以用组合数学公式直接秒杀。在 408 机试或面试中，通常要求掌握动态规划的空间优化解法。

解题思路一：动态规划（推荐）

  1. 状态定义： 设 dp[i][j] 为到达第 i 行第 j 列的不同路径数量。
  2. 状态转移方程： 因为只能向右或向下走，所以到达 (i, j) 的路径只能从它上面 (i-1, j) 或左边 (i, j-1) 走过来。 因此：dp[i][j] = dp[i-1][j] + dp[i][j-1]。
  3. 初始化边界： 网格的第一行（只能一直向右走）和第一列（只能一直向下走）的所有格子的路径数都是 1。即 dp[0][j] = 1，dp[i][0] = 1。
  4. 空间优化（降维）： 既然当前行的状态只依赖于上一行和当前行的前一个元素，我们可以只用一个一维数组 dp[n] 来滚动刷新状态：dp[j] = dp[j] + dp[j-1]，从而将空间复杂度从 O(m×n) 降到 O(n)。

解题思路二：组合数学（进阶） 机器人从左上角走到右下角，总共需要移动 m+n−2 步。其中，无论怎么走，都必须有 m−1 步是向下的，n−1 步是向右的。 因此，这就是一个经典的组合问题：在总共 m+n−2 步中，挑出 m−1 步向下的走法。 路径总数即为：Cm+n−2m−1​。只需写一个计算组合数的循环即可做到 O(m) 的时间复杂度与 O(1) 的空间复杂度（注意防溢出）。

🎯 带有权值的动态规划（教学重点）

这道题是上一道《不同路径》(Unique Paths) 的进阶版。在原来的纯路线计数基础上，加入了**“权值”**的概念，是考验能否灵活运用状态转移方程的绝佳题目。

解题思路：

  1. 状态定义： 设 dp[i][j] 为从左上角到达第 i 行第 j 列时的最小路径和。

  2. 状态转移方程： 同样地，因为只能向右或向下走，到达 (i, j) 的前一步必然是从上面 (i-1, j) 或左边 (i, j-1) 过来的。我们要找最小和，所以要在这两条路线中挑一个总和较小的，再加上当前格子的值： dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]。

  3. 初始化边界条件（极为重要）：

    • 起点：dp[0][0] = grid[0][0]
    • 第一行（只能从左边走过来）：dp[0][j] = dp[0][j-1] + grid[0][j]
    • 第一列（只能从上面走下来）：dp[i][0] = dp[i-1][0] + grid[i][0]

  4. 空间优化（原地修改）： 如果题目允许修改原网格，我们可以直接把 grid 数组当成 dp 数组来用，将空间复杂度直接降为 O(1)！如果不允许修改原数组，也可以使用一维的滚动数组，将空间复杂度降为 O(n)。

🎯 二维动态规划（教学重点）

这道题是 408 机试和一线大厂面试中极其经典的二维动态规划（DP）代表作。学会这道题，就相当于拿下了所有“两个字符串比对”类型题目（如编辑距离、最长公共子串）的通用钥匙。

解题思路：

  1. 状态定义： 设 dp[i][j] 表示 text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列的长度。

  2. 状态转移方程（核心）： 我们在对比 text1 的第 i 个字符和 text2 的第 j 个字符时，只有两种情况：

    • 相等： 如果 text1[i-1] == text2[j-1]（注意 C 语言下标从 0 开始），那么这个字符一定属于公共子序列的一部分。 dp[i][j] = dp[i-1][j-1] + 1
    • 不相等： 如果这两个字符不相等，那么它们不可能同时出现在同一个公共子序列中。最长公共子序列必然是舍弃其中一个字符后的最优解。 dp[i][j] = max(dp[i-1][j], dp[i][j-1])

  3. 初始化边界条件： 当其中一个字符串为空时，公共子序列的长度必然是 0。 即对于所有的 i 和 j，有 dp[i][0] = 0 且 dp[0][j] = 0。

  4. 空间复杂度优化（进阶）： 观察状态转移方程可以发现，dp[i][j] 仅仅依赖于当前行和上一行的数据（左边、上边、左上角）。因此我们可以把二维的 dp 数组压缩成两个一维数组，甚至只用一个一维数组加上一个临时变量来完成滚动更新，将空间复杂度从 O(M×N) 优化为 O(min(M,N))。
