🎯 递推策略（教学重点）

错排问题是组合数学中的经典模型，具有非常固定的递推公式。假设 n 个元素的错排数为 D(n)：

  1. 基本边界： 当 n=1 时，无法错排，即 D(1)=0；当 n=2 时，只有一种交换方式，即 D(2)=1。

  2. 状态转移方程： 对于第 n 个元素，它可以放在前 n−1 个位置的任意一个（共有 n−1 种选择）。假设它放在了第 k 个位置，那么原本在第 k 个位置的元素有两种情况：

    • 放到第 n 个位置：剩下的 n−2 个元素进行错排，方案数为 D(n−2)。
    • 不放到第 n 个位置：这相当于剩下的 n−1 个元素进行错排，方案数为 D(n−1)。

因此，核心递推公式为：D(n)=(n−1)×(D(n−1)+D(n−2))。

🎯 贪心与数学逻辑（教学重点）

这道题的核心在于数学分析和贪心思想。要在数组中选出三个数使其乘积最小，实际上只有两种组合可能：

  1. 三个最小的数相乘（例如数组全为正数，或者全为负数时）。
  2. 一个最小的数（即绝对值最大的负数）与两个最大的数相乘（例如数组中有正有负，通过“负 × 正 × 正”得负来获取极小值）。

解题思路： 假设将数组按从小到大排序为 A1​,A2​,…,An​：

  • 组合一的乘积为：A1​×A2​×A3​
  • 组合二的乘积为：A1​×An−1​×An​ 最终的最小乘积即为这二者中的最小值：min(A1​×A2​×A3​,A1​×An−1​×An​)。

进阶优化： 并不需要对整个数组进行 O(NlogN) 的完整排序，只需在单次 O(N) 的遍历中，利用几个变量分别维护并找出数组的最小的三个数和最大的两个数即可。

🎯 栈的典型应用（教学重点）

这道题是数据结构中**栈（Stack）的绝对经典应用，也是考研 408 数据结构大题的常客。核心思想是利用栈“后进先出”（LIFO）**的特性来处理配对问题。

解题思路：

  1. 遍历字符串： 从左到右依次读取每个字符。

  2. 遇到左括号： (、[、{ 时，将其压入栈中。

  3. 遇到右括号： )、]、} 时，需要与栈顶元素进行匹配：

    • 如果栈为空，说明没有对应的左括号，直接返回无效。
    • 弹出栈顶元素，检查是否与当前右括号类型匹配（例如 ( 匹配 )）。如果不匹配，返回无效。

  4. 遍历结束： 如果栈为空，说明所有左括号都成功闭合，返回有效；如果栈不为空（例如输入 ((），说明有未闭合的左括号，返回无效。

进阶优化： 如果字符串的长度为奇数，它肯定无法完全配对，可以在一开始就直接判断并返回无效（剪枝操作）。

🎯 高精度加法（教学重点）

这道题是 408 机试和秋招面试中极其经典的**“高精度计算（大数运算）”**入门题。当数字大小超过 long long 的极限（大约 19 位数字）时，我们必须依靠字符串或数组来模拟人类手工进行竖式加法的过程。

解题思路：

  1. 双指针从后往前： 设置两个指针分别指向 num1 和 num2 的末尾（即个位），从右向左按位相加。
  2. 处理进位 (Carry)： 每次相加时，除了两个字符对应的数字外，还要加上上一位的进位 carry（初始为 0）。当前位的实际结果是 (n1 + n2 + carry) % 10，新的进位是 (n1 + n2 + carry) / 10。
  3. 补齐短字符串： 如果其中一个字符串已经遍历完，可以将其对应的数字当作 0 继续参与运算，直到另一个字符串也遍历完。
  4. 处理最高位进位： 两个字符串都遍历完后，如果 carry 还不为 0（比如 9+1=10），必须在结果的最前面再追加一个 1。
  5. 反转结果： 因为我们是从个位开始计算并依次保存的，所以最后得到的结果是反的，输出前需要将生成的字符串反转。

这道题是考察贪心思想的绝佳范例。不要被“跳跃”的各种组合情况吓到，也不需要用复杂的深度优先搜索（DFS）去枚举每一种跳法，那样必定会超时。

🎯 动态规划基础（教学重点）

这道题是学习**动态规划（Dynamic Programming）**的完美入门题。它的本质是一个斐波那契数列（Fibonacci sequence）。

解题思路：

  1. 定义状态： 假设 dp[i] 表示爬到第 i 阶楼梯的方法数。

  2. 寻找状态转移方程： 思考一下，你怎么才能到达第 i 阶？

    • 你只能从第 i-1 阶跨 1 步上来。
    • 或者从第 i-2 阶跨 2 步上来。 所以，到达第 i 阶的方法数，就等于到达第 i-1 阶的方法数加上到达第 i-2 阶的方法数。即：dp[i] = dp[i-1] + dp[i-2]。

  3. 初始化边界条件：

    • 爬 1 阶只有 1 种方法：dp[1] = 1
    • 爬 2 阶有 2 种方法：dp[2] = 2

  4. 空间复杂度优化（滚动数组思想）： 在计算 dp[i] 时，我们只需要用到 dp[i-1] 和 dp[i-2] 这两个数字，不需要把之前所有的结果都存下来。因此，可以用两个变量 prev1 和 prev2 来交替滚动更新，把空间复杂度从 O(N) 降维到 O(1)。

🎯 动态规划与组合数学（教学重点）

这道题是爬楼梯（Climbing Stairs）的二维升级版。它既可以用动态规划求解，也可以用组合数学公式直接秒杀。在 408 机试或面试中，通常要求掌握动态规划的空间优化解法。

解题思路一：动态规划（推荐）

  1. 状态定义： 设 dp[i][j] 为到达第 i 行第 j 列的不同路径数量。
  2. 状态转移方程： 因为只能向右或向下走，所以到达 (i, j) 的路径只能从它上面 (i-1, j) 或左边 (i, j-1) 走过来。 因此：dp[i][j] = dp[i-1][j] + dp[i][j-1]。
  3. 初始化边界： 网格的第一行（只能一直向右走）和第一列（只能一直向下走）的所有格子的路径数都是 1。即 dp[0][j] = 1，dp[i][0] = 1。
  4. 空间优化（降维）： 既然当前行的状态只依赖于上一行和当前行的前一个元素，我们可以只用一个一维数组 dp[n] 来滚动刷新状态：dp[j] = dp[j] + dp[j-1]，从而将空间复杂度从 O(m×n) 降到 O(n)。

解题思路二：组合数学（进阶） 机器人从左上角走到右下角，总共需要移动 m+n−2 步。其中，无论怎么走，都必须有 m−1 步是向下的，n−1 步是向右的。 因此，这就是一个经典的组合问题：在总共 m+n−2 步中，挑出 m−1 步向下的走法。 路径总数即为：Cm+n−2m−1​。只需写一个计算组合数的循环即可做到 O(m) 的时间复杂度与 O(1) 的空间复杂度（注意防溢出）。

🎯 二维动态规划（教学重点）

这道题是 408 机试和一线大厂面试中极其经典的二维动态规划（DP）代表作。学会这道题，就相当于拿下了所有“两个字符串比对”类型题目（如编辑距离、最长公共子串）的通用钥匙。

解题思路：

  1. 状态定义： 设 dp[i][j] 表示 text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列的长度。

  2. 状态转移方程（核心）： 我们在对比 text1 的第 i 个字符和 text2 的第 j 个字符时，只有两种情况：

    • 相等： 如果 text1[i-1] == text2[j-1]（注意 C 语言下标从 0 开始），那么这个字符一定属于公共子序列的一部分。 dp[i][j] = dp[i-1][j-1] + 1
    • 不相等： 如果这两个字符不相等，那么它们不可能同时出现在同一个公共子序列中。最长公共子序列必然是舍弃其中一个字符后的最优解。 dp[i][j] = max(dp[i-1][j], dp[i][j-1])

  3. 初始化边界条件： 当其中一个字符串为空时，公共子序列的长度必然是 0。 即对于所有的 i 和 j，有 dp[i][0] = 0 且 dp[0][j] = 0。

  4. 空间复杂度优化（进阶）： 观察状态转移方程可以发现，dp[i][j] 仅仅依赖于当前行和上一行的数据（左边、上边、左上角）。因此我们可以把二维的 dp 数组压缩成两个一维数组，甚至只用一个一维数组加上一个临时变量来完成滚动更新，将空间复杂度从 O(M×N) 优化为 O(min(M,N))。

无重复字符的最长子串 (Longest Substring Without Repeating Characters)

解决这道题的核心算法是滑动窗口（Sliding Window）配合哈希表（或数组）。

你可以想象一个可以伸缩的窗口，用左右两个指针（left 和 right）框住一段没有任何重复字符的子串。

核心思路如下：

  1. 初始化： 准备两个指针 left = 0 和 right = 0，代表窗口的左右边界。准备一个变量 max_len 记录最长长度。再准备一个哈希表（或大小为128的布尔数组）来记录窗口内出现过的字符。

  2. 扩大窗口（右指针移动）： right 指针主动向右移动，每次考察一个新字符。

  3. 判断重复：

    • 如果新字符不在窗口内： 将其加入哈希表，计算当前窗口长度（right - left + 1），如果比 max_len 大就更新 max_len。
    • 如果新字符已经在窗口内： 说明出现了重复字符。此时，你需要开始缩小窗口。

  4. 缩小窗口（左指针移动）： 移动 left 指针，把它经过的字符从哈希表中剔除出去，直到把那个与新字符重复的字符也剔除出窗口为止。

  5. 继续循环： 缩小窗口后，新字符就可以合法进入窗口了。继续移动 right 指针，重复上述过程，直到 right 遍历完整个字符串。

一个简单的例子（以字符串 “abcabcbb” 为例）：

  • right 一直走到 “abc”，窗口内是 a, b, c，长度为 3。
  • right 走到第二个 a 时，发现窗口里有 a 了。
  • left 开始移动，把第一个 a 踢出窗口。此时窗口变成 b, c，然后把新的 a 加进来，窗口变成 b, c, a。
  • 以此类推，全程维护的最大长度就是答案。

“快乐数”定义如下：

  1. 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
  2. 然后重复这个过程直到这个数变为 1，也可能是无限循环但始终变不到 1。
  3. 如果经过这个过程最终结果为 1，那么这个数就是快乐数。

如果 n 是快乐数就返回 1 (true)，否则返回 0 (false)。

示例 1： 输入： n = 19 输出： 1 解释： 12+92=82 82+22=68 62+82=100 12+02+02=1

示例 2： 输入： n = 2 输出： 0

解析与 C 语言实现

核心思路：快慢指针 (Floyd 判圈算法) 题目明确指出，如果不是快乐数，数字的变换会陷入无限循环。这等价于经典的“链表找环”问题。

  1. 把求取下一个数字的操作视为链表的 next 指针。
  2. 设置两个变量：慢指针 slow 和快指针 fast。慢指针每次更新一次（走一步），快指针每次连续更新两次（走两步）。
  3. 如果 n 是快乐数，快指针最终会首先到达 1。
  4. 如果 n 不是快乐数，序列会进入环，快慢指针最终必定会在环内的某个数字相遇。

复杂度分析：

  • 时间复杂度：O(logn)。计算下一位平方和的操作次数与数字的位数成正比，不论是最终到达 1 还是陷入循环，其迭代步数都是对数级别的。
  • 空间复杂度：O(1)。仅使用了 slow 和 fast 两个整型变量记录状态，没有使用哈希表来存储历史记录。

题目：找出最短和最长的字符串

题目描述： 输入多行字符串，请按照原文本中的顺序输出其中最短和最长的字符串，如果最短和最长的字符串不止一个，请全部输出。

输入描述： 输入包括多行字符串，字符串的长度 len (1≤len≤1000)。

输出描述： 按照原文本中的顺序输出其中最短和最长的字符串，如果最短和最长的字符串不止一个，请全部输出。

解析与 C 语言实现

核心思路：两次遍历

  1. 数据存储与第一次遍历：因为不知道总共有多少行，且需要按原顺序输出，必须使用二维数组（或指针数组）将所有输入的字符串保存下来。在读取的同时，动态记录下出现过的最短长度 min_len 和最长长度 max_len。
  2. 处理换行符：使用 fgets 读取整行字符串时，会将行末的换行符 \n 也读入，需要将其替换为字符串结束符 \0 以保证长度计算准确。
  3. 第二次遍历与输出：为了分类清晰（通常此类OJ系统的要求），先遍历一遍数组输出所有长度等于 min_len 的字符串，再遍历一遍数组输出所有长度等于 max_len 的字符串。

题目：字典序最小的拼接字符串

题目描述： 给定一个字符串数组，请将数组中的所有字符串拼接起来，要求拼接后的最终字符串字典序最小。

输入格式： 一个包含若干个字符串的数组（或多行字符串输入）。

输出格式： 一个字符串，表示所有字符串拼接后字典序最小的结果。

示例 1： 输入： ["c", "b", "a"] 输出： "abc"

示例 2： 输入： ["ba", "b"] 输出： "bab" (解释：如果按普通字典序排序，"b" 在 "ba" 前面，拼接结果为 "bba"。但 "bab" < "bba"，所以应该把 "ba" 排在 "b" 前面。)

解析与 C 语言实现

核心思路：自定义排序规则 这道题的陷阱在于不能直接按照字符串原本的字典序进行排序。 必须比较两个字符串拼接后的结果： 设有两个字符串 s1 和 s2：

  1. 若 s1 + s2 < s2 + s1，则在排序结果中，s1 应该排在 s2 的前面。
  2. 若 s1 + s2 > s2 + s1，则 s2 应该排在 s1 的前面。
  3. 若相等，则两者的相对顺序无所谓。

通过这种自定义的比较规则对整个数组进行排序，最后将排序后的数组按顺序拼接，即可得到全局字典序最小的字符串。

OpenClaw（“龙虾”）智能体面试题

1. 核心机制与定位

  • 问题：OpenClaw 作为一个 Agent 框架，与传统的纯对话大语言模型（如网页端的 ChatGPT）在运行机制上最本质的区别是什么？
  • 考察点：传统大模型是被动问答工具，而 OpenClaw 是本地优先（Local-First）的超级助理。它能够接管系统权限，直接调用操作系统 API，自主执行复杂任务（如读写文件、部署服务器、发送邮件等），实现了从“给出方案”到“落地执行”的跨越。

2. 确定性工作流 (Lobster Engine)

  • 问题：OpenClaw 内置了名为 “Lobster” 的工作流引擎。为什么在复杂的自动化任务中，我们需要 Lobster 这种确定性管道（Deterministic Pipelines），而不是完全放权给大模型（LLM）去动态编排所有步骤？
  • 考察点：完全依赖 LLM 进行复杂编排会导致 Token 消耗剧增且极易引发“幻觉”。Lobster 提供了类型化的运行时环境、断点续传和强制审批节点（Approval Gates），使得多步工具调用更加稳定、可控且成本更低。

3. 安全防御与权限隔离

  • 问题：鉴于 OpenClaw 拥有极高的本地执行权限，企业在内网部署“龙虾”时，应如何防范其带来的系统穿透或供应链攻击风险？
  • 考察点：考察安全基线意识。包括：遵循最小权限原则（如创建专用的受限账户 clawuser 使用 rbash）、禁用高危系统命令、在独立沙箱或隔离网段中运行、以及对高风险操作强制接入人类审批（Human-in-the-loop）机制。

4. 多智能体协同 (Multi-Agent)

  • 问题：在 OpenClaw 生态中，如果需要配置两只“龙虾”——一只负责编写代码，另一只负责代码审查（Code Review），系统是如何实现这种协同交互的？
  • 考察点：了解 OpenClaw 的原生通信机制，例如通过 agentToAgent 进行节点间通信，以及基于 Session Key 的地址路由和 Webhook 触发机制。
